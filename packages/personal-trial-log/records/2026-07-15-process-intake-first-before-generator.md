# 입력 수집기를 생성기보다 먼저 구현

## 관찰된 제작 프로세스

프로토타입 생성기로 바로 넘어가지 않고, 사용자의 아이디어를 확정 요구사항으로 바꾸는 입력 수집기를 먼저 구현했다. 사용자가 적게 말한 부분을 하네스가 추론하지 않고, 답변이 명시될 때까지 질문 목록으로 남기는 구조를 선택했다.

## 드러난 의사결정 기준

구현 편의보다 요구사항 확정성을 우선했다. 특히 입력 문장에 단서가 있더라도, 사용자가 별도 답변으로 확정하지 않은 항목은 요구사항으로 인정하지 않는 기준을 세웠다.

## 공식 설계와의 관계

이 기록은 개인 제작 프로세스 문서다. 공식 요구사항은 `src/intake` 구현과 README 사용법에 반영했다.

## 반복 가능성

앞으로도 생성기보다 게이트와 수집기를 먼저 구현하는 흐름이 반복될 가능성이 높다.

## 다음 관찰 포인트

입력 수집기가 충분히 엄격한지, 반대로 실제 사용자가 답변하기 불편할 정도로 질문이 많은지 확인해야 한다.
