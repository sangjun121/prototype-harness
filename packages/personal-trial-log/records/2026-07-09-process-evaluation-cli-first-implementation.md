# 평가 CLI부터 시작한 첫 구현

## 관찰된 제작 프로세스

작성자는 프로토타입 생성기보다 평가 시스템을 먼저 구현하기로 한 계획을 문서로 남긴 뒤, 그 계획에 맞춰 첫 구현을 시작했다.

첫 구현은 외부 의존성 없이 Node.js 표준 라이브러리만 사용해 평가 입력 JSON을 읽고, 요구사항 게이트를 검사하고, 점수를 계산하고, 실패 유형을 분류하고, 실행 리포트를 생성하는 CLI로 잡았다.

## 드러난 의사결정 기준

- 생성 기능보다 평가 가능성을 먼저 확보한다.
- 처음부터 복잡한 프레임워크를 붙이지 않고 표준 라이브러리로 시작한다.
- 실행 결과는 사람이 읽을 수 있는 Markdown과 기계가 읽을 수 있는 JSON으로 함께 남긴다.
- 실행 산출물은 저장소에 커밋하지 않고 `runs/`로 분리한다.

## 공식 설계와의 관계

공식 설계의 "평가 게이트 우선" 원칙을 실제 코드 구조로 옮긴 첫 단계다.

이 구현은 아직 프로토타입 생성기가 아니라, 이후 생성기가 반드시 통과해야 할 평가 기반이다.

## 반복 가능성

높다.

앞으로도 새 기능을 만들기 전에 입력 형식, 게이트, 점수, 리포트 구조를 먼저 정의하는 방식이 반복될 가능성이 크다.

- 여러 실행 결과를 비교하는 trend summary를 어떻게 설계할지
